title: 装填配置
description: 装填配置
---

装填配置是插件的核心配置，用于指定插件在哪些网页上生效，以及如何将弹幕装填到视频上。

同一网址最多只能有一个装填配置生效，如果有多个装填配置匹配到同一个网址，插件会选择第一个匹配到的装填配置。

装填配置需要以下信息

## 名称

装填配置的名称，方便用户识别。可重名，但是不推荐。

## 视频元素

视频元素的选择器，用于指定视频元素。插件会将弹幕装填到这个元素上。

:::tip
一般情况下，默认的`video`选择器就可以满足大部分网页。

在网页有多个视频元素的情况（比如视频预览，广告等）可能需要指定一个更具体的选择器。
:::

## 适配

用来关联[自动模式](/details/auto-mode)。

## 网址（URL模式）

相当于一个白名单，只有在这些网址上插件才会生效。

准确的说，装填配置对应的网址不是单一的地址，而是一个[**模式**](https://developer.mozilla.org/en-US/docs/Web/API/URLPattern/URLPattern)，这个模式可以匹配多个网址。

拿B站为例，如果希望插件在B站上生效，可以使用`https://www.bilibili.com/*`这个模式，这个模式可以匹配所有的B站视频网址。

如果填写`https://www.bilibili.com/`，那么插件只会在B站首页生效。

如果填写`https://www.bilibili.com/video/*`，那么插件只会在视频播放页生效。

如果填写`https://*.bilibili.com/*`，那么插件会在所有的B站子域名下生效，包括动态，空间，直播等。

:::note
模式结尾必须包含`/`，比如`/`,`/*`,`/abc`,否则会提示格式错误。

同时模式必须是HTTPS的地址，不支持非HTTPS地址。
:::

最宽松的模式是`https://*/*`，这个模式可以匹配所有的HTTPS地址。

一般来说，如果希望匹配的网页是`https://www.example.com`，那么模式填写`https://www.example.com/*`或者`https://*.example.com/*`就可以满足需求。如果有特殊情况的请按照实际情况填写。

单个装填配置可以填写多个模式。

### `iframe`

有一个特殊情况是视频元素在`iframe`中，这种情况下需要填写`iframe`的地址，而不是网页本身的地址。

`iframe`的地址可以通过使用浏览器的开发者工具查看`iframe`元素的`src`属性获得。

原因见[工作原理](/details/limitations#iframe)。
