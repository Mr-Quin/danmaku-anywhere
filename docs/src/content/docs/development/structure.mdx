## 仓库地址

https://github.com/Mr-Quin/danmaku-anywhere

## 开发环境

- `Node.js` >= 22
- `pnpm` >= 10（请使用 `pnpm`，不要使用 `npm` 或 `yarn`）
- Chromium / Chrome（开发浏览器扩展时由 `dev:browser` 自动启动）

## 项目结构

本项目为`pnpm`多包`workspace`。

<FileTree>

- docs/ - 文档（本站，Astro / Starlight）
- app/
  - **web/** - Web应用，基于 Kazumi 规则的看番网站 (Angular)
- backend/
  - proxy/ - Cloudflare Worker，与弹弹play API交互、AI 服务、用户认证
- packages/
  - **danmaku-anywhere/** - 扩展主体 (React + Vite + @crxjs/vite-plugin)
    - src/
      - background/ - 后台脚本（Service Worker）
      - content/ - 内容脚本
        - controller/ - 控件
        - player/ - 弹幕播放器
      - popup/ - 弹出窗口
    - e2e/ - Playwright e2e 测试与开发浏览器启动脚本
  - danmaku-converter/ - 弹幕格式解析与归一化
  - danmaku-engine/ - 弹幕渲染器
  - danmaku-provider/ - 与第三方弹幕源 API 交互
  - dango/ - 声明式弹幕源 manifest 引擎
  - dango-manifests/ - 内置 dango manifest（JSON）
  - web-scraper/ - 网页/媒体元数据抓取
  - bangumi-api/ - Bangumi API 类型与 schema
  - result/ - `Result<T, E>` 类型工具
  - integration-policy/ - 站点集成策略 schema
- package.json

</FileTree>

## 安装

<Steps>

1. 安装依赖
   ```bash
   pnpm install
   ```
2. 构建共享包（扩展、Web 应用、Worker 都依赖于已构建的 `packages/*`）
   ```bash
   pnpm build:packages
   ```
</Steps>

<Aside type="tip">
跨 worktree 开发时，每个 worktree 都需要单独的 `node_modules/`，请在每个 worktree 中执行 `pnpm install && pnpm build:packages`。
</Aside>

## 开发

### 浏览器扩展

扩展位于 `packages/danmaku-anywhere`。

<Tabs>
  <TabItem label="Chrome">

    推荐使用 `dev:browser`，会自动启动一个独立的 Chromium 实例，预先加载扩展、启用扩展开发者模式、并将扩展固定在工具栏上：

    ```bash
    pnpm dev:browser
    ```

    HMR 已启用，源码改动会自动刷新。用户配置保存在 `packages/danmaku-anywhere/.playwright-profile`，关闭浏览器即结束开发会话。

    构建产物位于 `packages/danmaku-anywhere/dev/chrome`。

    若需手动加载：先运行 `pnpm dev`，然后在 `chrome://extensions` 启用开发者模式并加载已解压的扩展（指向 `dev/chrome`）。

    在开发模式下，扩展的弹窗与内容脚本右上角会显示一个红色 `DEV · <branch>` 水印，标明当前 worktree 的分支，方便多 worktree 并行开发时区分。

  </TabItem>
  <TabItem label="Firefox">
    Firefox开发需要`web-ext`，每次改动后需要重新构建

    用`--watch`选项运行构建命令
    ```bash
    pnpm build:firefox --watch
    ```

    另开一个终端运行
    ```bash
    pnpm start:firefox
    ```

  </TabItem>
</Tabs>

  </TabItem>

### Web App

Web 应用位于 `app/web`，依赖浏览器扩展才能完整工作。

```bash
cd app/web
pnpm start
```

### Backend (Cloudflare Worker)

后端位于 `backend/proxy`。

```bash
cd backend/proxy
pnpm dev
```

## 常用脚本

仓库根目录提供整套 monorepo 级脚本，优先使用而不是 ad-hoc 命令：

| 任务 | 命令 |
|---|---|
| 安装依赖 | `pnpm install` |
| 构建所有 | `pnpm build` |
| 只构建包 | `pnpm build:packages` |
| Lint + 自动修复 | `pnpm lint` |
| 类型检查 | `pnpm type-check` |
| 所有测试 | `pnpm test` |
| 格式化 | `pnpm format` |
| 提取 i18n key | `cd packages/danmaku-anywhere && pnpm i18n extract` |

Lefthook 会在 `git commit` 前对暂存文件执行 Biome 检查并自动修复格式。如果提交失败，先查看 Biome 输出。
